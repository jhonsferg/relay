## The Problem

RFC 3986's `url.ResolveReference()` treats "/" in a path as an **absolute reference**, which replaces the entire base path. This is correct for document servers but breaks API endpoints with path components.

### Example

```go
base := "https://api.example.com/v1"
path := "/Products"
// Expected: https://api.example.com/v1/Products
// Actual:   https://api.example.com/Products  ❌ Lost /v1!
```

### Why This Happens

RFC 3986 Section 5.3 defines path resolution as:

> If the reference path begins with "/", it is a reference to an absolute path and we use it as-is.

This works for relative paths between documents but breaks for API endpoints:
- Document servers: base `https://example.com/docs/` + path `/manual` → replaces docs
- API servers: base `https://api.com/v1` + path `/users` → should append to /v1

## The Solution

Relay uses a **dual-path strategy**: detect whether the base URL looks like an API endpoint, then apply the appropriate resolution strategy.

### Smart Detection

Detects common API patterns:
- Version prefixes: `/v1`, `/v2`, `/v3`, `/v4`, `/v5`
- Service patterns: `/api`, `/odata`, `/rest`, `/graphql`, `/soap`, `/sap`, `/data`, `/service`, `/services`
- Multi-segment paths: Any path with 2+ slashes

### Strategy Selection

**RFC 3986 Path** (for host-only URLs):
```go
base := "https://api.example.com"
path := "/users"
// Resolves to: https://api.example.com/users ✅
```

**Safe Path Normalization** (for API endpoints):
```go
base := "https://api.example.com/v1"
path := "/users"
// Detects /v1, uses safe normalization
// Resolves to: https://api.example.com/v1/users ✅
```

isAPIBase("https://api.example.com/v1")     // true

## Best Practices

### 1. Always Use Trailing Slashes for API Bases
```go
// ❌ Bad: missing trailing slash
client := relay.NewClient("https://api.example.com/v1")

// ✅ Good: Relay auto-normalizes, but explicit is clearer
client := relay.NewClient("https://api.example.com/v1/")
```

### 2. Keep Paths Relative Without Leading Slash
```go
// ❌ Confusing: leading slash creates absolute path
resp, _ := client.Get("/users").Execute()  // Actually works, but confusing

// ✅ Clear: explicitly relative
resp, _ := client.Get("users").Execute()   // More semantic
```

### 3. Use PathBuilder for Complex Paths
```go
// ❌ String concatenation is error-prone
path := "/api/v1/users/" + userID + "/posts"

// ✅ FluentPathBuilder is clearer and safer
path := relay.NewPathBuilder("/api/v1").
    Add("users", userID).
    Add("posts").
    String()
```

### 4. Configure Explicitly When Needed
```go
// Default: Auto-detection (recommended)
client := relay.NewClient(url)

// Explicit when auto-detection might fail
client := relay.NewClient(url).
    WithURLNormalization(relay.NormalizationAPI)
```

## FAQ

### Q: Will this break my existing code?
**A:** No. Smart detection is 100% backward compatible. All existing code continues to work without changes.

### Q: What if I have a host-only URL like "https://example.com"?
**A:** Smart detection recognizes this pattern and uses RFC 3986 resolution (which is correct for host-only URLs).

### Q: What if my API has an unusual path like "/svc/api/v1"?
**A:** Smart detection recognizes multi-segment paths (2+ slashes). If detection fails, use explicit configuration:
```go
client := relay.NewClient(url).
    WithURLNormalization(relay.NormalizationAPI)
```

### Q: How do I debug URL resolution?
**A:** Use `ResolveTest()` for debugging:
```go
result := relay.ResolveTest(&config, baseURL, path)
fmt.Printf("Resolved: %s\n", result.URL)
fmt.Printf("Strategy: %s\n", result.Strategy)
```

### Q: Does this affect streaming?
**A:** No. URL resolution behavior is identical for all request types (streaming, standard, batch, etc.).

### Q: What about fragments and query parameters?
**A:** Auto-normalization only adds trailing slashes to the domain+path component. Query parameters and fragments are preserved correctly.

## Technical Deep Dive

### Smart Detection Algorithm

```go
func isAPIBase(urlStr string) bool {
    // Parse URL and get path
    u, _ := url.Parse(urlStr)
    path := u.Path
    
    // Check for common API patterns
    if strings.HasPrefix(path, "/api") ||
       strings.HasPrefix(path, "/v1") ||
       // ... other patterns
       return true
    }
    
    // Check for multi-segment paths (2+ slashes)
    segments := strings.Count(path, "/")
    return segments > 1
}
```

### Strategy Selection

```go
func (r *Request) build(ctx context.Context) error {
    // 1. Detect if API base
    if isAPIBase(r.client.config.baseURL) {
        // 2. Use safe normalization
        return r.buildWithStringsBuilder(ctx)
    }
    
    // 3. Use RFC 3986 resolution
    return r.buildWithURLResolve(ctx)
}
```

### Allocation Optimization

**Before**: Created slice on every call
```go
apiPatterns := []string{"/api", "/v1", ...}  // 1 alloc
for _, pattern := range apiPatterns {
    if strings.HasPrefix(path, pattern) {
        return true
    }
}
```

**After**: Direct string comparisons
```go
if strings.HasPrefix(path, "/api") ||
   strings.HasPrefix(path, "/v1") ||
   // ... 0 allocations
   return true
}
```
